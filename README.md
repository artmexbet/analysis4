# analysis4

## Задание
Выполнение практической работы предполагает решение следующий задач:
1. Выполнить предварительную обработку данных
2. Построить граф для выбранного бизнес-процесса
3. Провести анализ полученного графа на предмет наличия в нем узких мест, циклов, отклонений и оптимальных путей
4. Предложить варианты оптимизации выбранного бизнес-процесса

## Установка и запуск

```bash
pip install -r requirements.txt
python process_mining_analysis.py
```

## Описание решения

### Бизнес-процесс: Обработка заявок в службе поддержки

Анализируется процесс обработки заявок, состоящий из следующих активностей:
- **Получение заявки** — регистрация входящей заявки
- **Классификация** — определение типа и приоритета заявки
- **Эскалация** — передача сложных заявок вышестоящему менеджеру
- **Обработка** — выполнение работ по заявке
- **Возврат на доработку** — возврат заявки для дополнительной обработки
- **Закрытие** — завершение работы с заявкой

### Результаты анализа

#### 1. Предварительная обработка данных
- Общее количество событий: 53
- Количество уникальных кейсов: 10
- Количество уникальных активностей: 6
- Пропущенные значения: 0

#### 2. Граф бизнес-процесса
- Узлов (активностей): 6
- Рёбер (переходов): 8

Основные переходы:
- Получение заявки → Классификация: 10 раз
- Обработка → Закрытие: 9 раз
- Классификация → Обработка: 6 раз
- Обработка → Возврат на доработку: 5 раз
- Классификация → Эскалация: 4 раза

#### 3. Анализ графа

**Узкие места:**
- Активность "Обработка" является узким местом (3 входящих связи)

**Циклы:**
- Обнаружен цикл: Обработка → Возврат на доработку → Обработка

**Отклонения от стандартного процесса:**
- Стандартный путь: Получение заявки → Классификация → Обработка → Закрытие (40% кейсов)
- Отклонения: эскалация (20%), возврат на доработку (30%), комбинация (10%)

**Оптимальный путь:**
- Получение заявки → Классификация → Обработка → Закрытие (3 шага)

**Анализ длительности:**
- Минимальная длительность: 2 часа
- Максимальная длительность: 30 часов
- Средняя длительность: 9 часов
- Медиана: 5 часов

Кейсы с аномально высокой длительностью (>7.8 ч) характеризуются наличием эскалаций и возвратов на доработку.

#### 4. Варианты оптимизации

1. **Устранение циклов** — улучшить качество первичной обработки (эффект: -20-30% времени)
2. **Оптимизация эскалации** — расширить полномочия специалистов первого уровня (эффект: -15-25% времени)
3. **Автоматизация классификации** — внедрить ML-классификацию (эффект: -50-70% времени классификации)
4. **Параллелизация** — начинать обработку на этапе классификации (эффект: -10-15% времени)
5. **Балансировка нагрузки** — перераспределить задачи между специалистами
